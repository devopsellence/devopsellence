# Contributing

Thanks for your interest in devopsellence.

## Current policy

- Bug reports, docs corrections, and product feedback are welcome through GitHub issues.
- Unsolicited code contributions are not currently accepted.
- Please do not open pull requests with code changes unless we have agreed on the work in advance.

## Why

devopsellence is open source, but the project is still early and the contribution policy is intentionally narrow while the product, roadmap, and commercial structure are still being defined.

## If you want to collaborate

Open an issue first and describe:

- the problem
- the proposed change
- why it should live in the public monorepo

If we want to move forward on code changes, we can agree on scope and any required contribution terms before implementation starts.

## Development notes

If you are working on code by prior agreement:

- keep changes small and reviewable
- use `mise` for toolchains and tasks
- run the relevant tests before handing work back
